# Android 设备指纹 SDK 受限数据采集指南

在现代 Android 生态系统（特别是 Android 10/API 29 及更高版本）中，Google 为了保护用户隐私，对许多底层硬件标识符和敏感信息的采集施加了严格限制。

本篇文档详细说明了 SDK 中**无法直接采集**或**受到严格限制**的数据项及其原因。

---

## 1. 核心标识符限制 (Android 10+)

自 Android 10 起，非系统应用无法获取不可重置的设备标识符。

| 受限数据项 | 限制说明 | 原因/政策 | 现状/影响 |
| :--- | :--- | :--- | :--- |
| **IMEI / MEID** | 无法获取 | Google 隐私政策 (Android 10+) | 调用返回 `null` 或抛出 `SecurityException` |
| **IMSI** | 无法获取 | Google 隐私政策 (Android 10+) | 无法唯一标识 SIM 卡用户 |
| **设备序列号 (SN)** | 无法获取 | Google 隐私政策 (Android 10+) | 调用 `Build.getSerial()` 将报错 |
| **MAC 地址 (Wi-Fi)** | 始终返回固定值 | Android 6.0 开始限制，10.0 彻底封死 | 始终返回 `02:00:00:00:00:00` |
| **MAC 地址 (蓝牙)** | 始终返回固定值 | Android 10.0 彻底封死 | 始终返回 `02:00:00:00:00:00` |

### 💡 替代方案：
*   **Android ID (SSAID)**：在应用未卸载前通常保持不变，是目前最常用的设备标识。
*   **OAID (匿名设备标识符)**：中国信通院推出的标识符，适用于国内主流厂商（华为、小米、OPPO 等）。
*   **UUID / GUID**：应用级生成的唯一标识，存储在 `SharedPreferences` 或外部存储中。

## 2. 权限受限数据 (需要用户授权)

以下数据项在技术上可以采集，但必须在 `AndroidManifest.xml` 中声明权限，并在运行时获得用户允许。

| 数据项 | 所需权限 | 采集风险 |
| :--- | :--- | :--- |
| **基站信息 (LAC/CID)** | `ACCESS_FINE_LOCATION` | 属于敏感地理位置信息，用户拒绝率高。 |
| **GPS 经纬度** | `ACCESS_FINE_LOCATION` | 属于高风险隐私，不建议在纯指纹 SDK 中强制要求。 |
| **附近 Wi-Fi 列表** | `ACCESS_WIFI_STATE` & 定位权限 | Android 11+ 要求扫描 Wi-Fi 必须同时开启定位开关。 |
| **SIM 卡数量/状态** | `READ_PHONE_STATE` | 需要动态权限，可能引起系统弹窗提醒。 |

## 3. 运行环境风险检测限制

| 数据项 | 限制说明 | 解决方案建议 |
| :--- | :--- | :--- |
| **模拟定位检测** | 必须配合位置服务使用 | 需从 `Location` 对象中检查 `isFromMockProvider()`。 |
| **Xposed/Hook 检测** | 存在对抗性 | 无法 100% 确定，需通过 JNI 层检查内存 Maps 镜像。 |
| **VPN 检测** | 仅能检测当前状态 | 通过 `ConnectivityManager` 检查，无法追溯历史。 |

---

## 4. 总结与合规性建议

1.  **最小化采集**：仅采集实现功能所必需的数据，并在隐私政策中向用户明示。
2.  **动态降级**：当权限被拒绝或 API 受限时，SDK 应能通过其他非敏感字段（如 CPU 架构 + 屏幕 DPI + 内置存储容量等）生成“模糊指纹”进行兜底。
3.  **合规合规合规**：建议集成 **OAID SDK**，以弥补 Android 10 以后无法获取 IMEI 的缺陷。
